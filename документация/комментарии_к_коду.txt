# Комментарии к коду — PDFYandexVisionBlock

## Файл: src/__init__.py

### Общая архитектура

Код организован в виде конвейера (pipeline):

  PDF-файл → Base64 → Yandex Vision API → JSON-ответ → Парсинг текста → Извлечение реквизитов → Структурированный результат

### Основные компоненты

1. **Классы данных (dataclasses)** — строки 162–228
   - `ImportantData` — ключевые реквизиты (номер, дата, ИНН, суммы)
   - `ItemData` — одна позиция из таблицы товаров/услуг
   - `DetailsData` — список позиций + краткое описание
   - `ExtractedDocument` — полный результат распознавания с методом `to_dict()`

2. **Работа с Yandex Vision API** — строки 273–368
   - `_request_payload()` — формирует JSON-запрос для batchAnalyze
   - `_post_batch_analyze()` — отправляет запрос, обрабатывает ответы
   - Автоматический подбор авторизации: Api-Key → Bearer → обмен OAuth на IAM
   - Fallback: если DOCUMENT_RECOGNITION не поддерживается, переключается на TEXT_DETECTION

3. **Извлечение текста из ответа API** — строки 371–481
   - `_extract_full_text_from_document_recognition()` — парсит структуру DOCUMENT_RECOGNITION
   - `_extract_text_detection_lines()` — парсит структуру TEXT_DETECTION
   - `_walk_strings()` — универсальный обход JSON для сбора текстовых фрагментов

4. **Извлечение реквизитов с помощью регулярных выражений** — строки 484–965
   - `_find_document_type()` — определяет тип документа (8 типов)
   - `_find_document_date()` — находит дату с учётом контекста
   - `_find_payment_order_number()` — номер платёжного поручения
   - `_find_total_amount()` — сумма документа
   - `_extract_counterparties()` — контрагенты (поставщик/покупатель)
   - `_extract_vat_and_discounts()` — НДС и скидки
   - `_extract_items_from_tables()` / `_extract_items_from_text()` — табличные позиции

5. **Обработка ошибок** — строки 1006–1077
   - `_show_error_message()` — кроссплатформенное диалоговое окно ошибки
   - Каждый тип ошибки (авторизация, квота, сеть, файл) перехватывается отдельно
   - При ошибке возвращается корректная структура с `"ok": false`

6. **Главная функция `run_pdf_yandex_vision()`** — строки 1008–1077
   - Декорирована `@window_logger` и `@log_decorator` для интеграции с Puzzle RPA
   - Валидирует входные параметры
   - Запускает весь конвейер обработки
   - Возвращает результат в выбранном формате (dict или JSON-строка)

### Регулярные выражения

Для извлечения данных используется 25+ скомпилированных регулярных выражений (строки 60–139).
Они покрывают:
- 8 типов документов (платёжное поручение, счёт-фактура, накладная, акт и т.д.)
- ИНН (10 или 12 цифр) и КПП (9 цифр)
- Роли контрагентов (поставщик, покупатель, плательщик)
- Суммы в различных форматах (с пробелами, запятыми, точками)
- Даты в формате ДД.ММ.ГГГГ
- Номера документов

### Кроссплатформенность

Функция `_show_error_message()` работает на:
- **macOS** — через AppleScript (`osascript`)
- **Windows** — через WinAPI (`ctypes.windll.user32.MessageBoxW`)
- **Linux** — через `tkinter.messagebox`

# Кейс 6 — Блок распознавания реквизитов PDF через Yandex Vision

**Команда:** Peacekamyhkin  
**Puzzle RPA Studio:** версия 3.0.3  
**Python:** 3.11

---

## Описание решения

Пользовательский блок (расширение) для **Puzzle RPA Studio**, который:

1. Принимает PDF-файл (в том числе отсканированный).
2. Отправляет его в **Yandex Vision OCR** через REST API (`batchAnalyze`, функция `DOCUMENT_RECOGNITION`).
3. Извлекает из ответа **структурированные реквизиты**:
   - тип документа (платёжное поручение, счёт-фактура, накладная, акт, договор и др.);
   - номер и дата документа;
   - сумма (с НДС, без НДС, сумма НДС);
   - контрагенты (название, ИНН, КПП);
   - табличные позиции (наименование, количество, цена, сумма).
4. Возвращает результат в виде **словаря Python (`dict`)** или **JSON-строки** — на выбор пользователя.
5. **Корректно обрабатывает ошибки** и показывает пользователю понятное системное окно:
   - пустой или неверный токен;
   - файл не найден;
   - превышена квота Yandex Vision;
   - нет доступа к сети / сервис недоступен.

## Входные параметры блока

| Параметр | Тип | Описание |
|---|---|---|
| **Токен** | Строка | OAuth-токен Yandex Cloud (или API-ключ сервисного аккаунта) |
| **Каталог (Folder ID)** | Строка | Идентификатор каталога Yandex Cloud |
| **Путь к файлу** | Строка | Абсолютный путь к PDF-файлу |
| **Язык** | Выпадающий список | `смешанный` / `русский` / `английский` |
| **Формат** | Выпадающий список | `Словарь (dict)` / `JSON-строка` |

## Инструкция по запуску

### Предварительные требования
- Puzzle RPA Studio **3.0.3**
- Python **3.11** (устанавливается вместе со Студией)
- Доступ к интернету
- Действующий **OAuth-токен** или **API-ключ** Yandex Cloud
- **Folder ID** каталога с активированным сервисом Yandex Vision

### Установка расширения

**Шаг 1.** Откройте терминал в корневой папке проекта (`Peacekamyhkin_кейс_6`) и установите зависимости:
```bash
pip install -r requirements.txt
```

**Шаг 2.** Скопируйте папку `решение/PDFYandexVisionBlock` в директорию расширений Puzzle RPA:
- **macOS:** `~/.puzzle_cache/code-storage-py-extensions/`
- **Windows:** `%USERPROFILE%\.puzzle_cache\code-storage-py-extensions\`

**Шаг 3.** Перезапустите Puzzle RPA Studio.

**Шаг 4.** Блок **«Распознать PDF (Yandex Vision)»** появится в категории **«Обработка документов» → «OCR»**.

> **Примечание:** В целях совместимости, файл `requirements.txt` вынесен из папки расширения. Это сделано намеренно, чтобы Puzzle RPA Studio не блокировала установку расширения красным сообщением об ошибке из-за зависимостей. Зависимости внутри кода подгружаются лениво (lazy import).
### Использование в роботе

1. Перетащите блок на холст.
2. Подключите к входам строковые переменные с токеном, Folder ID и путём к PDF.
3. Выберите язык и формат вывода.
4. Результат блока — это переменная, содержащая словарь (или JSON-строку) с извлечёнными реквизитами.
5. Используйте стандартные блоки Puzzle RPA для дальнейшей обработки данных.

## Технические детали

- **API:** Yandex Vision OCR (`batchAnalyze` → `DOCUMENT_RECOGNITION`, fallback на `TEXT_DETECTION`)
- **Авторизация:** автоматический подбор схемы (`Api-Key` → `Bearer` → обмен OAuth на IAM-токен)
- **Парсинг:** 25+ регулярных выражений для извлечения реквизитов из распознанного текста
- **Таблицы:** извлечение табличных данных из структуры ответа API и из текста
- **Обработка ошибок:** кастомные исключения + системные диалоговые окна (macOS: AppleScript, Windows: WinAPI `MessageBoxW`)

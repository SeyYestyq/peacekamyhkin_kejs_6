# Кастомный блок: Распознать PDF (Yandex Vision)

Данный проект представляет собой пользовательский блок (расширение) для PuzzleRPA, который использует Yandex Vision API (OCR) для распознавания PDF-документов (счетов-фактур, платежных поручений и др.) и извлечения структурированных данных.

## Что делает этот блок?

Блок принимает на вход PDF-файл, распознает текст и извлекает следующие данные в формате структурированного JSON или словаря:

*   **Важное (Реквизиты):**
    *   ИНН, КПП, Расчетный счет плательщика и получателя
    *   Название банков, БИК, Корреспондентский счет
    *   Суммы (с НДС, без НДС, сумма НДС), скидки
    *   Сумма прописью, Вид операции, Очередность платежа, УИН
    *   Номер и дата договора, счета-фактуры
*   **Данные:**
    *   Товары и Услуги (список позиций с названием, количеством, ценой и суммой)
    *   Краткое описание документа (например, "ПЛАТЕЖНОЕ ПОРУЧЕНИЕ №89332 от 14.03.2022")
*   **Метаданные:**
    *   Тип документа
    *   Номер и дата
    *   Путь к исходному файлу

## Установка и настройка (ПОШАГОВАЯ ИНСТРУКЦИЯ)

### Шаг 1: Получите доступ к Yandex Cloud

1.  Перейдите на [console.cloud.yandex.ru](https://console.cloud.yandex.ru)
2.  Создайте каталог (или используйте существующий)
3.  Получите **API-ключ**:
    *   Перейдите в `IAM → Ключи доступа → Создать ключ`
    *   Скопируйте полученный ключ
4.  Скопируйте **Folder ID** каталога (находится в общих настройках каталога)
5.  Включите API **Yandex Vision** в каталоге (если не включено)

### Шаг 2: Установите блок в PuzzleRPA

#### Вариант A: Через менеджер расширений (рекомендуется)
1.  Откройте PuzzleRPA Studio
2.  Перейдите в `Менеджер расширений` или `Настройки`
3.  Добавьте блок из репозитория или установите напрямую

#### Вариант B: Ручная установка
1.  Скопируйте всю папку `PDFYandexVisionBlock` в каталог расширений студии

### Шаг 3: Настройте параметры блока

При добавлении блока на схему заполните поля:

1.  **Токен** — ваш API-ключ от Яндекс Облака
2.  **Каталог (Folder ID)** — ID вашего каталога в Yandex Cloud
3.  **Путь к файлу** — полный путь к PDF-файлу (например, `/Users/user/Documents/invoice.pdf`)
4.  **Язык** — `смешанный` (рекомендуется для русских документов)
5.  **Формат** — `Словарь (dict)` для дальнейшей обработки в роботе

### Шаг 4: Подключите зависимости (библиотеки)

Блок автоматически установит необходимые библиотеки при первом запуске:
- `PyPDF2>=3.0.0` — для чтения PDF файлов
- `requests>=2.28.0` — для HTTP запросов к API

Если библиотеки не установились автоматически:
```bash
pip install PyPDF2 requests
```

## Частые проблемы и их решение (Troubleshooting)

---

### ❌ Ошибка: `ModuleNotFoundError: No module named 'src'`

**Причина:** Студия закэшировала расширение некорректно — файл `__init__.py` оказался в корневой папке вместо папки `src/`.

**Как исправить (быстро — выполните в Терминале):**

```bash
mkdir -p ~/.puzzle_cache/code-storage-py-extensions/PDFYandexVisionBlock/src
cp ~/.puzzle_cache/code-storage-py-extensions/PDFYandexVisionBlock/__init__.py \
   ~/.puzzle_cache/code-storage-py-extensions/PDFYandexVisionBlock/src/
```

**ИЛИ (ручная инструкция):**

1.  Закройте PuzzleRPA Studio полностью
2.  Откройте Finder → Перейдите в папку → Нажмите `Cmd + Shift + G`
3.  Вставьте путь: `~/.puzzle_cache/code-storage-py-extensions/PDFYandexVisionBlock`
4.  Создайте папку `src` (если её нет)
5.  Переместите файл `__init__.py` в папку `src`
6.  Перезапустите Studio

**Как исправить (полная очистка кэша):**

1.  Удалите папку `~/.puzzle_cache/code-storage-py-extensions/PDFYandexVisionBlock`
2.  Перезапустите PuzzleRPA Studio
3.  Студия заново скачает расширение с корректной структурой

---

### ❌ Ошибка: `'ItemData' object has no attribute '__dict__'`

**Причина:** Ошибка в коде расширения (несовместимость с dataclass slots).

**Как исправить:**
Обновите блок до последней версии или переустановите расширение.

---

### ❌ Ошибка: `Неверный токен или недостаточно прав`

**Причина:** Неправильный API-ключ или IAM-токен.

**Как исправить:**
1.  Проверьте, что токен скопирован полностью (без пробелов в начале/конце)
2.  Убедитесь, что в Yandex Cloud включен API Yandex Vision
3.  Проверьте, что Folder ID соответствует каталогу с включенным Vision API

---

### ❌ Данные не передаются в следующий блок

**Причина:** Следующий блок ожидает другой формат данных.

**Как исправить:**
* Используйте формат `JSON-строка`, если следующий блок работает с текстом
* Используйте формат `Словарь (dict)`, если нужно обрабатывать данные в роботе

---

### ❌ Пустые поля в результате

**Причина:** PDF не распознан корректно или документ нестандартного формата.

**Как исправить:**
1.  Проверьте PDF-файл — откройте его и убедитесь, что текст читаемый (не скан)
2.  Попробуйте изменить язык распознавания
3.  Для сканов используйте другой блок или предварительную обработку изображения

## Советы по использованию

1.  **Для платежных поручений** — блок хорошо извлекает реквизиты, суммы, ИНН
2.  **Для счетов-фактур** — дополнительно извлекает таблицу товаров
3.  **Для сканов** — может потребоваться предварительная обработка PDF
4.  **Тестирование** — начните с простого PDF-файла, затем переходите к сложным
